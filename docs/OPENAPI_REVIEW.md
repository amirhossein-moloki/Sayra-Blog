# Swagger / OpenAPI Documentation Review

This review evaluates the current state of the API documentation generated by `drf-spectacular`.

---

## Validation Summary

### 1. Schema Standardization
- **Success:** The project successfully uses `StandardizedAutoSchema` to wrap all successful responses (2xx) in a consistent `{data, messagesList, pagination}` envelope.
- **Success:** Enum name collisions (e.g., `status` for both Posts and Comments) have been resolved via `ENUM_NAME_OVERRIDES` in `settings.py`.

### 2. Authentication Definitions
- **Status:** JWT Bearer authentication is correctly defined globally.
- **Status:** The `GoogleLoginView` correctly indicates it accepts an `id_token` in the request body.

---

## Identified Gaps & Inconsistencies

### ⚠️ Missing Body Schemas
- **`publish_post` Endpoint:** Historically missed a request body schema. This was identified in the API Audit and fixed using `@extend_schema(request=None)`.
- **Media Upload:** Ensure `multipart/form-data` schemas are correctly represented for file fields in the `MediaCreateSerializer`.

### ⚠️ Undocumented Parameters
- **Dynamic Fields:** The `?fields=` query parameter, handled by `DynamicSerializerViewMixin`, is not always explicitly documented in the OpenAPI UI for all endpoints.

### ⚠️ Schema Mismatches
- **Jalali Dates:** API responses return Jalali formatted strings (e.g., `1403/03/22`), but the OpenAPI schema may still type them as standard ISO-8601 `date-time`. Clients should be notified of this discrepancy.

---

## Recommendations for OpenAPI 3.1
1. **Response Examples:** Add explicit `OpenApiExample` decorators to common endpoints (List Posts, User Me) to provide better developer onboarding.
2. **Error Schemas:** Define standard schemas for `400`, `401`, and `403` responses to ensure `messagesList` structure is well-defined in the documentation.
3. **Tagging:** Group endpoints by business domain (e.g., "CMS", "Identity", "Assets") rather than just application name.

---

## Generating the Latest Schema
To regenerate the documentation after code changes:
```bash
python manage.py spectacular --format openapi-json > openapi.json
```
Verify the output at `http://localhost:8000/api/schema/swagger-ui/`.
