# E2E Testing Guide

## 1. Introduction
This guide explains how to run and maintain End-to-End (E2E) tests for the Playnest application.

## 2. Test Structure
Tests are located in `tests/e2e/` and split into:
- **Logic Journeys**: `priority-journeys.spec.ts` (Supertest-based logic verification).
- **UI Journeys**: `cms/`, `payments/` (Playwright-based browser verification).

## 3. Environment Setup
Create a `.env.e2e` file with necessary test configurations (refer to `.env.example`).
Required variables:
- `DATABASE_URL` (Test DB)
- `REDIS_URL` (Test Redis)
- `JWT_ACCESS_SECRET`

## 4. Running Tests

### Logic Tests (Jest)
```bash
npm test tests/e2e/priority-journeys.spec.ts
```

### UI Tests (Playwright)
```bash
npx playwright test
```

## 5. Adding New Tests
1. **Logic**: Use Supertest to simulate full API flows.
2. **UI**: Use Playwright codegen (`npx playwright codegen`) to generate selectors for new SSR templates.

## 6. Known Limitations
- Playwright tests require a running instance of the app or use the `webServer` config in `playwright.config.ts`.
- External providers (SMS, Payment) are mocked in the E2E logic tests.
